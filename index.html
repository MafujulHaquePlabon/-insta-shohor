How does javascript works?

We know that, Javascript is the interpreted language but not a complied language.Javascript works in run time.an interpreter in the browser reads over the JavaScript code, interprets each line, and runs it. Javascript runs in such the browser with an engine called v8. When some JavaScript code comes into the browser or JavaScript source code comes into the browser, the browser parse them and make a syntax tree. V8 interpreter ignition generates bytecode from the syntax tree. But to execute javascript source code faster in browser, V8 engine converts javascript code into machine code with its own internal turbofan compiler just in time (JIT).

What does javascript event loop do?

JavaScript has a runtime model based on an event loop, which is responsible for running the code. Javascript runs asynchronous and synchronous. The stack moves synchronously but the queue moves asynchronous. function calls from a stack of frames. When the stack continues in a synchronous manner, the Queue works when the (LIFO) rules are completed. Queue first in first out (FIFO) rules.For example: like a train ticket line. There is also a memory heap where it keeps references to various things which it uses when needed and as garbage when not needed. The whole thing runs like a javascript loop. If a deley is given, it acts as a queue in an asynchronous way. Also acts as event loop blocking and unlocking.
